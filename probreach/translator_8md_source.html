1 ## Translator Usage
2 
3 The translator from *.pdrh* to *Simulink®/Stateflow®* models is available as a separate tool alongside ProbReach.
4 
5 ## Requirements
6 
7  - The packages already listed under [How to Build](https://github.com/dreal/probreach/blob/master/doc/build.md)
8 
9  - Currently only available for Linux, tested on Ubuntu 16.04, 17.10, 18.04 and Fedora 26
10 
11  - And a local installation of MATLAB (version 2017b or later) with Simulink and Stateflow addons installed,
12  as described [here](https://uk.mathworks.com/help/install/ug/install-mathworks-software.html).
13

42 ## Usage
43 To translate a given *.pdrh/.drh* model to Simulink run
44 ```
45 pdrh2simulink <path/to/model/file>
46 ```
47 For example,
48 ```
49 pdrh2simulink /home/user/repos/probreach/model/cars/collision.drh
50 ```
51 An optional ```--verbose``` flag can be passed to the command line tool to print additional translation details.
52 
53 Upon successful translation, a *.slx* file with a matching model name will be created
54 in the current directory. Note that since no valid Matlab
55 identifiers can contain dashes in their name, all dashes, "-", in the input files are replaced with underscores "_".
56 
57 ## Model Advice
58 
59 Some considerations need to be taken into account when developing models which are intended to be translated.
60 
61 Expressing strong equality ```x = y``` in jump guards is discouraged as the
62 translated Stateflow model might miss events (due to round-off errors, etc.) and the
63 respective jump will not occur. Instead, encode these as inequalities - ```>=``` or ```<=``` - for increasing and
64 decreasing values respectively. See the example below.
65 The translator cannot make any assumptions during execution about which inequality is to be used, leaving that
66 responsibility to the user at this stage
67 until the option to specify tolerance values for the translated comparisons is implemented.
68 
69 ```
70 ((v1 = 0))==>@4(and (s1' = s1) (s2' = s2) (v1' = v1) (v2' = v2) (v01' = v1) (v02' = v2) (tau' = 0));
71 ```
72 For a decreasing value of ```v1``` in the active mode the expression should become
73 ```
74 ((v1 <= 0))==>@4(and (s1' = s1) (s2' = s2) (v1' = v1) (v2' = v2) (v01' = v1) (v02' = v2) (tau' = 0));
75 ```
