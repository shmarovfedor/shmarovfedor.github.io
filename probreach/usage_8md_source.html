1 # File format
2 
3 We start with the example of a `.pdrh` file. See below the content of `stop-nonlinear.pdrh`.
4 ```
5 model: npha;
6 /*
7  * The model of a car deceleration scenario
8  */
9 #define v_max 55.56 // [200 km/h] maximum speed of a car (m/s)
10 #define v_100 27.78 // m/s
11 #define drag 3.028e-4 // some average value (1/m)
12 #define mu 1
13 #define alpha 0.05776
14 #define t_react 1.2
15 
16 [0,1000] s; // distance m
17 [0,v_100] v; // velocity m/s (16.67 m/s = 60 km/h)
18 [0,30] tau; // time in seconds
19 [0,30] time;
20 [4.0,6.0] a_d;
21 dist_normal(4,0.1) beta;
22 
23 // accelerating
24 {
25 mode 1;
26 flow:
27 d/dt[s]= v;
28 d/dt[v]= alpha*exp(-alpha*tau+beta)-drag*v*v;
29 d/dt[tau]= 1.0;
30 jump:
31 (and (v >= v_100))==>@2(and(s'=s)(v'=v)(tau'=0)(beta'=beta)(a_d'=a_d));
32 }
33 
34 // reacting
35 {
36 mode 2;
37 flow:
38 d/dt[s]= v;
39 d/dt[v]= -drag*v*v;
40 d/dt[tau]= 1.0;
41 jump:
42 (and (tau = t_react))==>@3(and(s'=s)(v'=v)(tau'=0)(beta'=beta)(a_d'=a_d));
43 }
44 
45 // braking
46 {
47 mode 3;
48 flow:
49 d/dt[s]= v;
50 d/dt[v]= -a_d-drag*v*v;
51 d/dt[tau]= 1.0;
52 jump:
53 (and (v <= 0))==>@4(and(s'=s)(v'=v)(tau'=0)(beta'=beta)(a_d'=a_d));
54 }
55 
56 // stopped
57 {
58 mode 4;
59 flow:
60 d/dt[s]= v;
61 d/dt[v]= 0.0;
62 d/dt[tau]= 1.0;
63 jump:
64 
65 }
66 
67 init:
68 @1(and (s = 0) (v = 0) (tau = 0));
69 
70 goal:
71 @4(and(tau=0)(s>=400));
72 ```

74 ## Model types
75 
76 **ProbReach** performs automatic identification of the model type. However, it is recommended that the user specifies the model type explicitly. The available model types are:
77 * `model:ha` - specifies a deterministic hybrid system (*i.e.* the one not featuring any parameters).
78 * `model:pha` - specifies a parametric hybrid system featuring only random (*e.g.* discrete and/or continuous) parameters.
79 * `model:npha` - specifies a parametric hybrid system featuring both random and continuous nondeterministic parameters.

81 ## Constants declaration
82 
83 There are two possible ways of declaring a constant in the model:
84 * Using a C preprocessor macro `#define <var> <value>`. This could also be used to define functions
85 * Alternative way of defining a constant is `[<value>]<var>;`
86 
87 ## Variable declaration
88 
89 In order to define a variable or a continuous nondeterministic parameter one must provide its name and range as `[<value>,<value>]<var>;`.
90 
91 ## Random parameter declaration
92 
93 The continuous nondeterministic parameters are declared in the same way as variables. The difference between a variable and a parameter in the model is that every variable has dynamics associated with further in the modes definition as opposed to the parameters.
94 
95 The random parameters are defined using a key word. The following distribution are available:
96 * `dist_normal(<value>,<value>)<var>;` - defines a normal random parameter `<var>` where the first `<value>` is the mean and the second `<value>` is the standard deviation of the distribution.
97 * `dist_uniform(<value>,<value>)<var>;` - defines a uniform random parameter `<var>` where the first `<value>` is the minimum and the second `<value>` is the maximum values of the uniform distribution.
98 * `dist_exp(<value>)<var>;` - defines an exponential random parameter `<var>` where the `<value>` is the rate value of the exponential distribution.
99 * `dist_discrete(<value>:<value>[,<value>:<value>])<var>;` - defines a discrete random parameter `<var>` where for each pair `<value>:<value>` the second `<value>` corresponds to the probability mass associated with the first `<value>`.
100 * `dist_gamma(<value>,<value>)<var>;` - defines a gamma random parameter `<var>` where the first `<value>` is the shape and the second `<value>` is the scale of the gamma distribution.
101 
102 Also it is possible to define continuous random parameters which probability density functions are nontrivial. Such parameter can be defined as:
103 * `dist_pdf(<fun>,<value>,<value>,<value>)<var>;` - defines a continuous random parameter `<var>` which probability density function (PDF) is defined by `<fun>`. The PDF must be written in the infix notation. The first `<value>` and the second `<value>` define the minimum and the maximum value of `<var>`. In case if a distribution has unbounded support the keywords `-infty` and `infty` can be used. The fourth `<value>` is needed when the distribution has unbounded support. It defines a point from where the integration will be performed in both directions in order to enclose the defined unbounded random variable. For more information about how the unbounded continuous random parameters are handled see [reference]. For example, a normal random variable `x` with the mean `1.0` and standard deviation `0.1` can be defined using `dist_pdf` as: `dist_pdf((1/(0.1*sqrt(2*3.14159265359)))*exp(-(x-1)*(x-1)/(2*0.1*0.1)),-infty,infty,1.0) x;`.

105 ## Defining modes
106 
107 Every mode defines a discrete state of the system. The continous dynamics are defined inside the mode.
108 ```
109 {
110 mode <id>;
111 
112 flow:
113 d/dt[<var1>] = <ode1>;
114 ...
115 d/dt[<varN>] = <odeN>;
116 
117 jump:
118 <guard1>==>@<next_id1><reset1>;
119 ...
120 <guardM>==>@<next_idM><resetM>;
121 }
122 ```
123 * `mode <id>;` - defines the `<id>` of the mode.
124 * `flow:` - starts definition of the continuous dynamics of the system. It is followed by the list of ODEs in the form `d/dt[<var>] = <ode>;` where `<var>` is the name of the variable and `<ode>` is the ODE associated with `<var>`. All ODEs should be written in the infix notation.
125 * `jump:` - starts definition of the discrete transitions from the current state. It is followed by the list of discrete transitions in the form `<guard>==>@<next_id><reset>;` where `<guard>` defines the condition when the system may take a transition to mode `<next_id>` setting the initial value of the continuous dynamics in mode `<next_id>` as specified in `<reset>`.
126  * each `<guard>` - is the quantifier free first order formula. The best way of doing this is to write `<guard>` in the disjunctive normal form as `(or (and (<expr> <op> 0) ... ) ... (and (<expr> <op> 0) ... ))` where each `expr` is a mathematical expression written in infix notation and each `<op>` is a comparison operator from the list `{<,<=,>,>=,=}`.
127  * each `<reset>` - is the quantifier free first oreder formula in the form `(and (<var>'=<expr>) ...)` where `<var>'` is the initial value of `<var>` in mode `<next_id>` and `expr` is a mathematical expression written in infix notation.
128 
129 ## Defining the initial state
130 
131 The initial state of the system is defined using the keyword `init:` followed by `@<id>(and (<var>=<expr>) ...)` where `<id>` is the id of the initial mode and each `(<var>=<expr>)` specifies the initial value of the variable `<var>`. Currently, **ProbReach** supports a single initial state.
132 
133 ## Defining the goal state
134 
135 The goal state of the system is defined using the keyword `goal:` followed by `@<id>(or (and (<expr> <op> 0) ... ) ... (and (<expr> <op> 0) ... ))` where `<id>` is the id of the goal mode and `(or (and (<expr> <op> 0) ... ) ... (and (<expr> <op> 0) ... ))` is the quantifier free formula in disjunctive normal form. Currently, **ProbReach** supports a single goal state.

140 ## Command Line Options
141 In order to see the list of `<solver-options>` run `dReal --help`. The following options are available in `ProbReach`.
142 
143 #### General options:
144 
145  -h/--help - displays help message
146  -k <int> - defines the reachability depth (default: the shortest path length if exists)
147  -l <int> - defines the reachability depth lower bound (cannot be used without -u; default: the shortest path length if exists)
148  -u <int> - defines the reachability depth upper bound (cannot be used without -l; default: the shortest path length if exists)
149  -t <int> - number of CPU cores (default: maximum number of CPU cores)
150  --verbose - outputs computation details (default false)
151  --verbose-result - outputs the runtime and the number of samples (statistical model checking only; default false)
152  --version - displays current version of the tool
153 
154 #### Solver related options:
155 
156  --delta-sat - uses the delta-sat answer of dReal to conclude about satisfiability of the evaluated formula (statistical model checking and hybrid automata only; default false)
157  --solver <path> - full path to the solver
158 
159 #### Special options:
160 
161  --time-var-name <string> - the name of the variable representing time in the model (default tau)
162 
163 #### Statistical model checking options:
164 
165  --bayesian-acc <double> - half-length of the confidence interval in Bayesian estimations (default: 0.01)
166  --bayesian-conf <double> - confidence value in Bayesian estimations (default: 0.99)
167  --cross-entropy - enables Cross-Entropy algorithm
168  --cross-entropy-term-arg <double> - termination argument (variance) for Cross-Entropy algorithm (default: 0.01)
169  --chernoff-acc <double> - half-length of the confidence interval in Chernoff-Hoeffding method (default: 0.01)
170  --chernoff-conf <double> - confidence value in Chernoff-Hoeffding method (default: 0.99)
171  --elite-ratio <double> - defines the fraction of the sample size - elite samples which are used in Cross-Entropy algorithm for updating the distribution parameters.
172  --min-prob - computes confidence interval for the minimum reachability probability (default: maximum reachbility probability)
173  --sample-size <int> - number of sample per iteration of Cross-Entropy algorithm
174 
175 #### Formal verification options:
176 
177  -e/--precision-prob <double> - length of the probability enclosure (default: 0.001)
178  --integral-inf-coeff <double> - ratio for the continuous random variables with unbounded support (default: 0.1)
179  --integral-pdf-step <double> - step value used for bounding domains of unbounded continuous random variables (default 0.1)
180  --partition-nondet - partitions the domain nondeterministic parameters up to the value defined in --precision-nondet (default: false)
181  --partition-prob - obtains a partition of the domain of continuous random parameters satisfying -e/--precision-prob (default: false)
182  --precision-nondet [<var> <double>] - defines the precision vector for the nondeterministic parameters
183  --precision-ratio <double> - used to define precision passed to the solver as "solver-precision = min-box-dimension * precision-ratio" (default: 0.001)
